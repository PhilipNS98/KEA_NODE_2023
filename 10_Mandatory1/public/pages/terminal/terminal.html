Terminal Commands

Here are some handy terminal commands to know in general and for Node projects:

To welcome you to Node JS and show the version:

node --version

To perform a soft kill:

^C

To perform a hard kill:

^D

To use the prompt that is an interpreter for Node commands:

>

To clear the terminal screen when in node:

console.clear()

To run code in Node:

node filename.js

Here are some additional commands that may be useful:

To list files and directories:

ls

To change directory:

cd directory_name

To create a new directory:

mkdir directory_name

To remove a file:

rm filename

To remove a directory:

rm -r directory_name

Types in Node

In Node, every value has a type, such as string, number, boolean, object, and function. The typeof operator can be used to determine the type of a value.

simply write 'typeof' followed by a value in the terminal and you will get the type in response.

Here are some examples of what you can check the type of and the response:

  • > typeof false

    'boolean'

  • > typeof 5.5

    'number'

  • > typeof function () {}

    'function'

  • > typeof "something"

    'string'

  • > typeof null

    'object'

  • > typeof []

    'object'

Type coercion is when JavaScript automatically converts a value from one type to another. This can happen implicitly, as in the case of adding a number and a string, or explicitly, using functions like Number() or String() to convert a value to a different type.

For example, the expression 2 + "2" coerces the number 2 into a string and concatenates it with the string "2", resulting in the string "22". The typeof operator will then return "string" for this expression.

In some cases, coercion can be unpredictable or even produce unexpected results. For instance, [] + {} returns the string "[object Object]", while {} + [] returns the number 0. This is because the + operator can mean both addition and concatenation, depending on the types of the operands involved, and the order in which they are evaluated.

It's important to understand how type coercion works in order to avoid bugs and ensure that your code behaves as expected.


Soft VS Hard Equality

In JavaScript, there are two types of equality operators: the '==' (soft equality) operator and the '===' (hard equality) operator. Soft equality will attempt to coerce the two operands to a common type before comparing them, while hard equality will only return true if the operands are of the same type and have the same value.

Here are some examples to illustrate the difference between soft and hard equality in Node:

// Soft equality console.log(2 == "2"); // true console.log(true == 1); // true console.log(null == undefined); // true // Hard equality console.log(2 === "2"); // false console.log(true === 1); // false console.log(null === undefined); // false

In the first set of examples, the soft equality operator is used to compare two values that are of different types. The operator coerces the two values to a common type before comparing them, resulting in 2 == "2" returning true, since both values are considered equivalent when coerced to a string.

In the second set of examples, the hard equality operator is used to compare the same values. Since the values are of different types, hard equality returns false, since the two values are not considered equivalent.

It's generally recommended to use hard equality over soft equality in most cases, since it avoids unexpected behavior caused by type coercion. However, there are some cases where soft equality may be useful or necessary, such as when comparing user input or when checking for null or undefined values.


Git Commands

  • Initializes a new Git repository in the current directory.

    > git init

  • Creates a copy of a remote repository on your local machine.

    > git clone [repository_url]

  • Displays the current state of the repository, including which files are modified and which files are staged.

    > git status

  • Adds a file to the staging area, which means it's ready to be committed.

    > git add [file_name]

  • Commits the changes in the staging area to the repository, with a message describing the changes.

    > git commit -m "[commit_message]"

  • Pushes the committed changes to the remote repository.

    > git push

  • Pulls changes from the remote repository and merges them into your local repository.

    > git pull

  • Lists all the branches in the repository.

    > git branch

  • Switches to a different branch in the repository.

    > git checkout [branch_name]

  • Merges the changes in a different branch into the current branch.

    > git merge [branch_name]
