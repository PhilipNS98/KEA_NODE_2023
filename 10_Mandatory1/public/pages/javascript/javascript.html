JavaScript Variables

In JavaScript, variables are used to store data values that can be accessed and manipulated throughout the program. There are a few rules to keep in mind when working with variables:

When declaring a variable using const or let, you must assign a value to it immediately:

const me = { name: "Philip" }; const hobbies = ["Football", "Swimming"]; hobbies.push("Cooking"); me.hobbies = hobbies; me.name = "Johannes";

Once a value is assigned to a variable using const, it cannot be reassigned:

// This will throw an error: "TypeError: Assignment to constant variable." // me = {};

Here is a table of variables and their difinition:

Variable Definition
let A keyword used to declare a block-scoped variable that can be reassigned.
const A keyword used to declare a block-scoped variable that cannot be reassigned.
var A keyword used to declare a function-scoped variable that can be reassigned.

Quotes

When working with strings, you can use either single or double quotes to enclose the string. To avoid having to use escape sequences, it's best to use the opposite quote inside the string. If you need to use both types of quotes, you can use backticks to enclose the string and then use the other quotes inside:

const hobbyOne = "Football skill level: '6'"; const hobbyTwo = 'Swimming skill level: "3"'; const hobbyThree = `Cooking skill level: "'${2 + 2}'"`; console.log(hobbyThree); // Output: "Cooking skill level: '4'"

Variable Scope

In JavaScript, variables can be declared with either var, let, or const. Using var can lead to unintended consequences, such as variable hoisting and variable scope issues. It's recommended to use let or const instead. If you need to declare a global variable, it's best to explicitly define it using the window object.

"use strict"; // This will throw a reference error since the variable is not defined. // totalGlobalVariable = "Never ever do this"; // Avoid using var for variable declaration. // var globalVariable = "Also never do this"; // Use let or const instead. { let someValue = true; { let someValue = false; } console.log(someValue); // Output: true } // To explicitly declare a global variable, use the window object. // window.globalVariable = "This is a global variable";

Iteration with setTimeout

When using a loop with setTimeout, it's important to use let instead of var to avoid scope issues:

// This will output the same value (6) six times. for (var i = 0; i <= 5; i++) { setTimeout(() => { console.log(i); }, 1000); } // This will output a value between 0-5 each time. for (let i = 0; i <= 5; i++) { setTimeout(() => { console.log(i); }, 1000); }


Loop Methods in JavaScript

Loop methods are higher-order functions in JavaScript that can help simplify and streamline the process of iterating through arrays. Rather than writing traditional loops, which can often be verbose and cumbersome, loop methods provide a more concise syntax for performing common operations on arrays.

Some common loop methods in JavaScript include:


'map'

'map()' method creates a new array with the results of calling a provided function on every element in the calling array.

const numbers = [1, 2, 3, 4, 5]; const mappedNumbers = numbers.map((num) => num * 2); console.log(mappedNumbers); // Output: [2, 4, 6, 8, 10]

'filter'

'filter()' method creates a new array with all elements that pass the test implemented by the provided function.

const arr = [1, 2, 3, 4, 5]; const filteredArr = arr.filter((num) => num > 3); console.log(filteredArr); // Output: [4, 5]

'find'

'find()' method returns the value of the first element in the array that satisfies the provided testing function.

const arr = [1, 2, 3, 4, 5]; const found = arr.find((num) => num > 3); console.log(found); // Output: 4

'reduce'

'reduce()' method executes a reducer function on each element of the array, resulting in a single output value.

const arr = [1, 2, 3, 4, 5]; const reduced = arr.reduce((accumulator, currentValue) => accumulator + currentValue); console.log(reduced); // Output: 15

'sort'

'sort()' method sorts the elements of an array in place and returns the sorted array.

const arr = [3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5]; const sortedArr = arr.sort(); console.log(sortedArr); // Output: [1, 1, 2, 3, 3, 4, 5, 5, 5, 6, 9]

'forEach'

'forEach()' method executes a provided function once for each array element

const arr = [1, 2, 3, 4, 5]; arr.forEach((num) => { console.log(num * 2); }); // Output: // 2 // 4 // 6 // 8 // 10

Using loop methods can help make your code more readable, maintainable, and performant, as they often allow you to accomplish the same tasks with fewer lines of code. However, it's important to note that loop methods are not always the best solution for every situation, and there may be cases where traditional loops or other approaches are more appropriate.


Common Array and String Methods in JavaScript

In JavaScript, arrays and strings are two of the most commonly used data types. There are a variety of methods available to manipulate and work with these data types. In this topic, we'll explore some of the most common methods available for arrays and strings in JavaScript.

toFixed

The 'toFixed()' method is used to format a number with a specific number of digits after the decimal point.

let num = 12.345; console.log(num.toFixed(2)); // Output: "12.35"

charAt()

The 'charAt()' method returns the character at a specified index in a string.

let str = "Hello World!"; console.log(str.charAt(6)); // Output: "W"

replace()

The 'replace()' method is used to replace a specified value with another value in a string.

let str = "Hello World!"; console.log(str.replace("World", "Universe")); // Output: "Hello Universe!"

push()

The 'push()' method is used to add one or more elements to the end of an array and returns the new length of the array.

let arr = [1, 2, 3]; arr.push(4); console.log(arr); // Output: [1, 2, 3, 4]

splice()

The 'splice()' method is used to add or remove elements from an array at a specified index.

let arr = [1, 2, 3]; arr.splice(1, 0, 4); console.log(arr); // Output: [1, 4, 2, 3]

pop()

The 'pop()' method is used to remove the last element from an array and returns that element.

let arr = [1, 2, 3]; let lastElement = arr.pop(); console.log(lastElement); // Output: 3 console.log(arr); // Output: [1, 2]

concat()

The 'concat()' method is used to merge two or more arrays and returns a new array.

let arr1 = [1, 2, 3]; let arr2 = [4, 5, 6]; let newArr = arr1.concat(arr2); console.log(newArr); // Output: [1, 2, 3, 4, 5, 6]

slice()

The 'slice()' method is used to extract a section of an array or string and returns a new array or string.

let arr = [1, 2, 3, 4, 5]; let slicedArr = arr.slice(2, 4); console.log(slicedArr); // Output: [3, 4] let str = "Hello World!"; let slicedStr = str.slice(0, 5); console.log(slicedStr); // Output: "Hello"

These are just a few of the most commonly used methods for arrays and strings in JavaScript. There are many more available that can help you manipulate and work with these data types.


Function Types

Functions are a crucial part of JavaScript and allow developers to write reusable code. There are different types of functions that can be used in JavaScript depending on the requirements of the application. In this section, we will discuss and demonstrate how to create and use different types of functions in JavaScript, including normal functions, anonymous functions, arrow functions, lambda functions, and callback functions.

Normal Functions:

A normal function is created using the function keyword followed by the function name, parameter(s), and function body enclosed in curly braces. Normal functions can be defined with a function name or assigned to a variable. They can be invoked using the function name or the variable name if the function is assigned to a variable.

function sum(a, b) { return a + b; } console.log(sum(2, 3)); // Output: 5 // Assigned to variable const diff = function(a, b) { return a - b; } console.log(diff(5, 2)); // Output: 3

Anonymous Functions:

Anonymous functions are functions that are defined without a name. They are commonly used as function expressions that are assigned to a variable or passed as an argument to another function. Anonymous functions can be invoked using the variable name that holds the function expression.

const multiply = function(a, b) { return a * b; } console.log(multiply(2, 3)); // Output: 6

Arrow Functions:

Arrow functions were introduced in ES6 as a shorthand for writing function expressions. They are defined using the '=>' symbol after the parameter list and can have an implicit or explicit return value. Arrow functions are often used in functional programming and provide a more concise way of writing functions.

const divide = (a, b) => { return a / b; } console.log(divide(6, 3)); // Output: 2 // Implicit return const multiplyByTwo = num => num * 2; console.log(multiplyByTwo(5)); // Output: 10

Lambda Functions:

Lambda functions are similar to arrow functions but have a slightly different syntax. They are also known as lambda expressions and can be defined with or without parameters.

const square = x => x * x; console.log(square(3)); // Output: 9 const message = () => console.log("Hello World!"); message(); // Output: Hello World!

Callback Functions:

A callback function is a function that is passed as an argument to another function and executed after the completion of the parent function. Callback functions are commonly used in event handling, asynchronous programming, and functional programming.

function greeting(name) { console.log(`Hello ${name}!`); } function processUserInput(callback) { const name = prompt("Please enter your name: "); callback(name); } processUserInput(greeting); // Output: Hello (name)!

All in all, functions are a powerful tool in JavaScript and can be used to make code more modular, reusable, and efficient.


Cross-Site Scripting (XSS)

Cross-site scripting (XSS) is a type of security vulnerability found in web applications. It allows attackers to inject malicious code (usually in the form of a script) into a legitimate website that other users then execute. This can lead to the theft of sensitive data such as cookies, session tokens, or other private information. In this topic, we will discuss the different types of XSS attacks and how to prevent them.


Types of XSS Attacks:

Type Definition
Reflected XSS: Reflected XSS occurs when an attacker sends a link containing a malicious script to a victim. When the victim clicks on the link, the script executes in the victim's browser, giving the attacker access to sensitive data. Reflected XSS can be prevented by sanitizing user input and properly validating all user-generated content.
Stored XSS: Stored XSS is a type of XSS attack where the malicious script is stored on the server, waiting for the victim to access the affected web page. This type of attack can be particularly dangerous because the script can affect multiple users. Stored XSS can be prevented by validating all user input and using proper encoding techniques.
DOM-Based XSS: DOM-based XSS occurs when the script is embedded in the DOM of the affected web page. This can happen when user input is not properly sanitized before being added to the DOM. DOM-based XSS can be prevented by sanitizing all user input and properly validating all user-generated content.

Prevention Techniques:

  1. Sanitize User Input: All user input should be sanitized to remove any special characters or scripts that could be used in an attack.
  2. Validate User-Generated Content: All user-generated content should be validated to ensure that it contains only safe and expected content.
  3. Use Proper Encoding Techniques: All user-generated content should be properly encoded to prevent XSS attacks.
  4. Use Content Security Policy (CSP): CSP is a security feature that allows web developers to specify which content can be loaded on their web pages. It can be used to prevent XSS attacks by only allowing trusted sources to be loaded.
  5. Use HttpOnly Cookies: HttpOnly cookies prevent malicious scripts from accessing session cookies, which can be used to steal sensitive information.

Here is an example of a vulnerable code snippet that could be used in an XSS attack:

var name = document.getElementById("name").value; document.write("Welcome, " + name + "!");

This code snippet is vulnerable because it uses user input without properly sanitizing it. An attacker could inject a script into the name field that would execute in the victim's browser when the code is executed.

To prevent XSS attacks, we could sanitize the input before using it:

var name = document.getElementById("name").value; name = name.replace( /</g, "<").replace(/>/g, ">"); document.write("Welcome, " + name + "!");

This code snippet uses regular expressions to replace any angle brackets in the input with their HTML entities, preventing any scripts from being executed.

Another way that attackers can exploit XSS vulnerabilities is by injecting malicious code into a website's HTML, such as by changing the innerHTML or innerText of an element.

For example, let's say that our website has a feature where users can submit comments. We might have a form where users can enter their comment text, and then we use JavaScript to add their comment to the page dynamically, like this:

<div id="comments"> <!-- existing comments here --> </div> <form> <label for="comment">Leave a comment:</label><br> <textarea id="comment" name="comment"></textarea><br> <button type="submit">Submit</button> </form> <script> const form = document.querySelector('form'); const comments = document.querySelector('#comments'); form.addEventListener('submit', event => { event.preventDefault(); const commentText = document.querySelector('#comment').value; // add the comment to the page const newComment = document.createElement('div'); newComment.innerHTML = commentText; comments.appendChild(newComment); }); </script>

However, an attacker could take advantage of the fact that we're using 'innerHTML' to inject their own malicious code into our page. For example, they could submit a comment like this:

<script> alert('This site has been hacked!'); </script>

When our JavaScript code adds this comment to the page using 'innerHTML', the attacker's code will be executed in the user's browser, displaying the alert message and potentially doing even worse things, like stealing user data or hijacking their session.

To avoid this type of attack, we should use 'innerText' instead of 'innerHTML' whenever we're inserting untrusted content into our page. This will ensure that any HTML tags are properly escaped and treated as text, preventing them from being executed as code.

So, we can modify our code above to use 'innerText' instead of 'innerHTML' like this:

<div id="comments"> <!-- existing comments here --> </div> <form> <label for="comment">Leave a comment:</label><br> <textarea id="comment" name="comment"></textarea><br> <button type="submit">Submit</button> </form> <script> const form = document.querySelector('form'); const comments = document.querySelector('#comments'); form.addEventListener('submit', event => { event.preventDefault(); const commentText = document.querySelector('#comment').value; // add the comment to the page using innerText const newComment = document.createElement('div'); newComment.innerText = commentText; comments.appendChild(newComment); }); </script>

Now, if an attacker tries to submit a comment with malicious code, it will be treated as text and displayed as such, preventing any code injection attacks.


Type: Module in package.json

In Node.js, modules can be imported using either the require() function or the import statement. The require() function uses the CommonJS module system, which is the default in Node.js. On the other hand, the import statement uses the ECMAScript module system, which was introduced in ES6.

Setting "type": "module" in package.json tells Node.js to use the ECMAScript module system for your project. This means that you can use the import statement to import modules instead of require().

For example, with "type": "module" in package.json, you can import an external module like this:

import express from 'express';

Without "type": "module", you would have to import the module like this:

const express = require('express');

Note that using the import statement requires the use of file extensions when importing local files. For example, you would need to use "./module.js" instead of "./module" when importing a local module with import.
